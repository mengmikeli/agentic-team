### 5. `toFixed()` Resilience

The change at `report.mjs:77` introduces a `v.costUsd != null` guard before calling `.toFixed(4)`. If `costUsd` were a non-numeric truthy value (e.g., a string), `toFixed()` would throw. However, STATE.json is machine-generated by the harness via `JSON.stringify`, so non-numeric `costUsd` values are not a realistic scenario. The same pattern exists at line 73 for `totalCostUsd.toFixed(4)` where the guard is `totalCostUsd != null`. Both are acceptable given the internal data source.

## Findings

üîµ bin/lib/report.mjs:51 ‚Äî `task.title` in "What Shipped" bullets is rendered without markdown escaping. If a task title contained `[text](url)`, it would render as a hyperlink in REPORT.md. Low risk: STATE.json is internally generated by the harness, not user-authored.

üîµ bin/lib/report.mjs:77 ‚Äî `v.costUsd.toFixed(4)` will throw `TypeError` if `costUsd` is present but non-numeric (e.g., string `"free"`). The `!= null` guard does not protect against this. Low risk: STATE.json is machine-generated; non-numeric `costUsd` is not a realistic input.
